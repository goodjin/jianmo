# Markly E2E 测试

本项目使用 Playwright 进行端到端测试。

## 安装依赖

```bash
npm install
npx playwright install chromium
```

## 运行测试

### 方式一：VS Code 扩展测试（推荐）

在开发模式下运行 VS Code 并加载扩展：

```bash
# 1. 打开项目
code .

# 2. 按 F5 启动扩展开发主机

# 3. 在调试控制台运行测试
```

### 方式二：Playwright UI 模式

```bash
npm run test:e2e:ui
```

### 方式三：命令行模式

```bash
npm run test:e2e
```

### 方式四：有头模式（显示浏览器）

```bash
npm run test:e2e:headed
```

## 测试结构

```
e2e/
├── markly.spec.ts    # 主测试文件
├── utils.ts          # 测试辅助工具
└── vscode-runner.ts # VS Code 运行器
```

## 测试覆盖的功能

| 测试用例 | 功能 |
|---------|------|
| 完整功能流程测试 | 打开→预览→编辑→格式化→切换→保存 |
| 预览界面渲染 | Milkdown 编辑器渲染 |
| 工具栏按钮功能 | 所有工具栏按钮点击 |
| 快捷键功能 | Cmd+B/I/Z 等 |
| 模式切换 | 源码/预览模式切换 |
| 大纲面板 | 大纲显示和跳转 |
| 图片预览 | 图片点击预览 |
| 表格功能 | 表格插入 |
| 代码块功能 | 代码块插入 |
| 保存功能 | 文件保存 |

## 测试策略

1. **单流程覆盖多功能**：完整功能流程测试按用户正常使用顺序测试多个功能
2. **独立测试**：每个功能有独立的测试用例，便于定位问题
3. **失败重试**：CI 环境下自动重试失败测试
4. **截图/录屏**：失败时自动保存截图和录屏

## 添加新测试

在 `e2e/markly.spec.ts` 中添加新的测试用例：

```typescript
test('新功能测试', async ({ vscode }) => {
  // 测试代码
});
```

## 常见问题

### 测试无法连接 VS Code

确保扩展已正确加载，可以在 VS Code 中打开任意 `.md` 文件验证。

### WebView 无法访问

某些测试需要访问 WebView 内部元素，确保在预览模式下进行测试。
