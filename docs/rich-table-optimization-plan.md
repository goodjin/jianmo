# Rich 表格优化与重构计划

> 状态：草案（待评审后按阶段执行）  
> 关联代码：`webview/src/plugins/markly-table-rich.ts`、`webview/src/components/MilkdownEditor.vue`、`webview/src/components/Toolbar.vue`、`webview/src/App.vue`、`e2e/ui-suite/markly-ui.test.js`

## 1. 目标

在 **Rich（Milkdown）模式** 下，让表格能力在 **可发现性**、**粘贴/互操作健壮性**、**工程化与回归** 三方面达到「可发布、可维护」标准，并与 GFM 序列化行为一致。

## 2. 范围（三类必须覆盖）

| 类别 | 说明 |
|------|------|
| **A. 体验 / 可发现性** | 工具栏、右键菜单、快捷键说明；避免「只有隐藏组合键」 |
| **B. 粘贴与矩阵 / 边界健壮性** | TSV/CSV/HTML 剪贴板策略、表头语义、大表与异常输入、与 `prosemirror-tables` / GFM 一致 |
| **C. 工程化 / 测试 / 重构** | 命令层收敛、E2E+单测、性能与安全边界、文档与版本 |

## 3. 阶段划分（建议顺序）

1. **阶段 0**：基线与现状冻结（快捷键、解析规则、已知限制写入本文档「附录」）。  
2. **阶段 1（A）**：工具栏 +（可选）右键菜单 + 用户可见的快捷键提示。  
3. **阶段 2（B）**：剪贴板策略统一（text/html/text/plain）、CSV 增强或策略收紧、表头/对齐、粘贴上限与降级。  
4. **阶段 3（C）**：命令模块收敛、E2E（含粘贴）、性能与边界单测、发版说明。

阶段 2 可与阶段 1 部分并行，但 **E2E 建议在阶段 2 之后一次性补全**，避免重复改脚本。

## 6. 风险与依赖

- **VS Code / Electron 剪贴板**：E2E 模拟 `text/html` 可能比 `text/plain` 难；C3 可分「必做 TSV」与「可选 HTML」。  
- **快捷键冲突**：Mod+Alt+方向键在部分系统/辅助功能下可能被占用；需在 A3 中说明并可配置（远期）。  
- **包体**：若 B2b 引入 CSV 库，需评估对 `App.js` 体积影响。

## 7. 附录：当前实现速查（执行本计划时请先核对）

以下以仓库内已实现行为为准，若代码变更请同步改本文档附录。

- **结构快捷键**（表格内）：Mod+Alt+↑/↓/←/→ 增行/列；Mod+Alt+Backspace 删行；Mod+Alt+Shift+Backspace 删列。  
- **表格内粘贴优先级（B1）**：在表格单元格内粘贴时，优先尝试 `text/html` 的 `<table>`，失败则回退到 `text/plain` 的 TSV/CSV 网格解析。  
- **CSV 策略（B2）**：多行/单行 CSV 采用“严格零依赖解析”（必须闭合引号、闭合后无垃圾字符、支持尾随逗号空字段、多行列数必须一致）；否则降级为普通粘贴。  
- **表头/对齐语义（B3）**：粘贴矩阵不会把“剪贴板首行”自动当表头；表头语义由目标表格结构决定。若落点在 header 单元格，则粘贴内容保持 header cell 类型。对齐属于列级结构属性，粘贴仅填充内容，不应重置对齐。  
- **相关文件**：`webview/src/plugins/markly-table-rich.ts`、`webview/src/components/MilkdownEditor.vue`、`e2e/ui-suite/markly-ui.test.js`。

### 7.1 边界手工清单（B5）

在 macOS/Windows/Linux 各至少 smoke 1 次（重点：Rich 模式 + 表格内）。勾选项均以“不会破坏表格结构、不会卡死、序列化回 GFM 正常”为通过标准。

- [ ] **删除边界**：光标在表格第一行（header）执行“删除当前行”，表格应仍可编辑且序列化不异常（建议保留至少 1 行表头语义或按编辑器既定规则降级）。  
- [ ] **删除边界**：光标在表格最后一行执行“删除当前行”，表格应仍可编辑。  
- [ ] **删除边界**：光标在表格最后一列执行“删除当前列”，表格应仍可编辑。  
- [ ] **最小表格**：只剩 1 行（或 1 列）时继续删行/删列的行为应可预期（拒绝/降级/删除整个表格三选一，实际行为需与 UX 一致）。  
- [ ] **跨区域粘贴**：在表格内粘贴一个超过上限的矩阵（行/列/总格），应降级为普通粘贴或拒绝，不应卡死。  
- [ ] **表格外粘贴**：光标在表格外粘贴 TSV/CSV/HTML table，应不触发“插入 cells”逻辑（按默认粘贴/默认建表规则走）。  
- [ ] **歧义文本**：在表格内粘贴普通多行段落（无 tab、列数不一致、坏 CSV），不应误解析成矩阵插入。  
- [ ] **HTML 优先级**：表格内粘贴同时含 `text/html` 与 `text/plain`（如 Excel），应优先 HTML table（若可解析），否则回退 plain。  
- [ ] **右键菜单**：表格内右键菜单可见，表格外右键不应出现表格菜单（或菜单禁用）；点击“下方插入行”等命令后结果与工具栏/快捷键一致。
