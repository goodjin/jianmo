# 复盘：中期 post-mortem（M351）

## 发生了什么（摘要）

中期阶段的核心工作是把“可用”推进到“可发布、可维护、可自救”：

- IR 实现层退出主路径
- 协议/诊断/自救口径固化
- i18n/A11y/E2E/包体预算/发布门禁系统化

## 做对了什么

- **门禁前置**：`preflight` 聚合门禁，减少“最后一刻爆炸”
- **自救优先**：统一诊断入口，支持场景收敛
- **预算治理**：bundle budget 让性能/包体变成可讨论的事实

## 做错了什么（或成本太高）

- 可选研究项在 roadmap 中太多，容易造成“看起来没做完”的错觉
- 许可/NOTICE 等合规项如果不提前排期，会在发布临界点变成阻塞

## 行动项（落到 Far Phase）

- 将合规（NOTICE/第三方许可）拉成独立迭代
- 把回归矩阵写入 release playbook，形成“每次必做”
- 继续削减大依赖或做更细粒度的代码拆分（特别是 webview vendor 体积）
