# M196–M198：2.0 风险清单与验收/决策

## 主要风险（示例清单）

1. **默认体验改变**：用户打开 `.md` 的默认编辑器被改变且不易回退  
2. **协议 breaking**：extension↔webview 消息不兼容导致空白/功能失效  
3. **序列化差异**：同一内容保存后 Markdown 发生不可逆变化  
4. **性能退化**：大文档/大表格卡顿边界回退  
5. **导出不可用**：PDF/HTML 导出在某些平台/网络环境失败率上升  
6. **安全风险**：外链资源策略不清晰导致合规问题  
7. **回滚困难**：用户升级后无法简单回退/恢复旧行为

## M197：验收剧本（可重复）

- **基础**
  - 打开 `.md` → Rich 可用（失败能降级 Source）
  - Rich/Source 切换光标与滚动不离谱
  - 保存后再打开：内容不漂移（round-trip fixtures）
- **表格**
  - TSV/HTML 粘贴
  - Tab/Enter/删除结构快捷键
- **图片**
  - 粘贴保存、缺失检测、批量修复、未引用清单
- **导出**
  - PDF/HTML 导出（含 Mermaid、表格、图片）
  - 导出失败诊断包可复制
- **AI（可选）**
  - 未开启时不出站
  - 开启后：错误提示清晰、不会连发

## M198：什么时候该升 2.0（触发条件）

满足任一条就应进入 2.0 评审：

- 移除 IR 且导致协议/默认体验 breaking
- 默认编辑器策略发生改变（例如强制关联、默认 Rich 且不可关闭）
- 保存/导出序列化对用户产生不可忽视的不兼容差异
