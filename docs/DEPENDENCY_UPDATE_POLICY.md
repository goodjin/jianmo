# M179：依赖更新策略（建议版）

目标：在不牺牲稳定性的前提下，持续获得安全修复与工具链收益。

## 1. 更新节奏

- **每 2–4 周一次**常规依赖更新（小步快跑）。
- 遇到 **安全公告/高危 CVE** 可随时单独热修。
- **`npm audit`**：团队侧建议 **高危在 7 日内**评估并修复或记录例外；中低危进入常规节奏（与 **M249** 对齐）。

## 2. 更新优先级

1. **安全修复**（优先）  
2. **构建/测试工具链**（例如 eslint/vitest/vite）：小版本优先，避免一次跨太大  
3. **编辑器内核相关**（Milkdown / ProseMirror / CodeMirror）：谨慎更新，必须配套回归门禁  

## 3. 风险评估清单（更新 PR 必填）

- 变更范围：仅 dev deps / 运行时 deps / both
- 关键链路是否覆盖回归：
  - Rich 启动/降级
  - 表格粘贴/Tab/Enter
  - 图片粘贴/缺失修复
  - 导出 PDF/HTML（含 Mermaid）
- `npm test` 是否通过
- 若涉及 VS Code UI 测：是否跑过 `npm run test:vscode:ui:stable`（有桌面环境时）

## 4. 版本锁定建议

- **扩展运行时依赖**：谨慎升级，必要时先开分支做试验再合入。
- **工具链依赖**：可常规升级，但要保证 CI/本机一致（Node 版本保持对齐）。
