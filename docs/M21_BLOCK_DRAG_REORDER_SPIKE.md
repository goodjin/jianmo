# M21：Rich 块级拖拽重排 — Spike 备忘（未实施）

## 目标

大纲级 **章节块** 拖拽重排已在侧栏部分支持；本 spike 指 **正文内** 大块（段落/标题/列表容器）在 Rich（ProseMirror）内 DnD。

## 技术候选

- **ProseMirror**：基于 `prosemirror-dropcursor` / 自定义 `dragstart` + `Slice` 序列化；需处理表格、列表嵌套与撤销合并。
- **与 Outline 复用**：谨慎合并数据源，避免「大纲拖一次、正文再拖一次」双真相。

## 结论（阶段记录）

在 **IR 冻结** 且 **Rich 为主线** 前提下，建议 **晚于** Rich/Source 等价清单中「列表/标题稳定性」项再立项；本仓库当前 **不启用** 正文块 DnD，以免与表格选区、图片拖拽冲突。
