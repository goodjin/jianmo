# 安全：漏洞修复与公告口径（M358）

## 结论（本阶段）

本阶段 **不引入额外安全修复 PR**（避免在封板后段引入回归），只固化：

- 如何快速定位风险
- 如何发布公告口径
- 如何回滚与自救

## 处理流程（建议）

1. **确认范围**：extension / webview / export / dependencies
2. **优先级**：
   - RCE / 任意文件写入 / 路径穿越：最高优先级
   - XSS（webview）与导出 HTML 注入：高优先级
   - DoS（超大输入卡死）：中优先级
3. **修复后门禁**：
   - `npm run preflight`
   - 必要时 `npm run test:vscode`
4. **公告口径**：
   - 说明影响版本范围
   - 说明是否可能泄露内容（默认应为否）
   - 给出用户缓解措施（升级、重载窗口、切 Source、自救中心）

## 用户缓解措施（模板）

- 立即升级到修复版本
- 执行 `Developer: Reload Window`
- 如遇异常，打开自救中心复制诊断信息并提交 issue
