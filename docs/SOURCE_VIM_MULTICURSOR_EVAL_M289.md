# Source：Vim / 多光标评估（M289，可选）

## 结论（建议）

- **多光标**：优先保持 **VS Code 原生多光标**（多选、列选择）在 Source 下可用；对 Webview 内 CodeMirror 的多光标不做额外承诺，仅保证不主动破坏常用快捷键。
- **Vim**：不默认引入 CodeMirror Vim 模式（体积、键位冲突、学习成本、测试矩阵会显著扩大）。若未来要做，建议：
  - 以 **配置开关**提供（默认 off）
  - 单独里程碑/单独门禁
  - 明确与 VS Code 原生 Vim 扩展的边界（谁接管键盘事件）

## 风险

- **快捷键冲突**：Vim 接管后可能与现有 `markly.*` 命令、查找替换、表格快捷键冲突。
- **测试成本**：需要新增一套行为回归（插入、撤销、查找、保存、切换模式）。

## 建议的下一步（若产品决定要做）

- 先做“冲突矩阵”与 FAQ（哪些键位以 VS Code 为准、哪些以编辑器为准）
- 再做最小 PoC：仅 Normal/Insert + 基础移动/删除，不做宏/寄存器等重功能
