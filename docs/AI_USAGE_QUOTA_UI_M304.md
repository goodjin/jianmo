# AI：用量/配额展示（M304，可选）

## 目标

在不引入计费/Token 估算复杂度的前提下，给用户与排障提供“AI 请求是否在工作、是否频繁失败”的可观测性。

## 本次落地（轻量版）

- Webview 侧在本地会话内统计 AI 请求：
  - 请求次数 `requests`
  - 成功次数 `success`
  - 失败次数 `failure`
  - 最近一次错误摘要 `lastError`
  - 最近一次发起请求时间 `lastRequestAt`
- 这些信息会进入“复制诊断信息”输出（diagnostics package），便于用户反馈问题时带上。

## 非目标（刻意不做）

- 不读取/展示真实额度（不同 provider 口径不一致，需要对接各家 API）
- 不估算 token（估算误差大、且会引入额外依赖与性能开销）
- 不写入持久化存储（避免隐私/历史污染；后续如需要可再评估）
