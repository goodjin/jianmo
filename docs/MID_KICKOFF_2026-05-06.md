## 兼容策略（体验优先）

- **默认策略**：尽量兼容（字段适配、容错、只要能编辑就让用户继续编辑）。
- **失败兜底**：无法兼容时，必须：
  - 自动切到 **Source**
  - 提示用户执行 **Reload Window** / 升级
  - 提供 **复制诊断**（含协商版本信息）
- **兼容窗口**：只承诺最近 1 个窗口（例如最近 N 个 minor）；过旧则提示升级，避免无限背债。

## 回滚策略（最重要的“安全阀”）

- 兼容层出问题时：允许通过配置开关或快速补丁回退到“强制降级 Source”（保证不白板）。
- IR 删除推进中：每个版本切片都保留可回退路径与门禁证据（`gates:stable`）。

## 工程门禁

- 必须：`npm run gates:stable`
- 发版前：`npm run check:release`
- 协议/类型：任何消息变更必须同步 `messageGuards` + `messageContract`。
