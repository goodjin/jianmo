## 1. 项目概述

### 1.1 背景与目标

**背景**:
简墨是一款基于 Milkdown/ProseMirror 框架开发的 VSCode Markdown 编辑器插件，目标是提供类似 Typora 的 WYSIWYG 编辑体验，同时保持 VSCode 的开发工作流集成。

**当前状态**:
- 已实现 CommonMark + GFM 核心语法支持
- 已实现基础的 WYSIWYG 编辑功能
- 已实现数学公式、高亮等扩展功能
- 已优化内容同步机制

**目标**:
通过对标 Typora 和 Office Viewer，完善编辑器功能，达到生产可用级别。

**对标参考**:
- 主要对标 **Typora**（桌面端 WYSIWYG Markdown 编辑器标杆）
- 参考 **Office Viewer**（VSCode 生态中最成熟的 WYSIWYG Markdown 编辑器）

### 1.2 目标用户

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| 技术写作者 | 撰写技术文档、博客的开发者 | 代码高亮、快捷键、VSCode 集成 |
| 内容创作者 | 撰写文章、笔记的用户 | WYSIWYG、流畅编辑、导出功能 |
| 知识管理者 | 使用 Markdown 管理知识库的用户 | 表格、图片、数学公式支持 |

### 1.3 核心价值主张

**VSCode 中的 Typora 体验**：在 VSCode 中获得媲美 Typora 的 WYSIWYG 编辑体验，无需切换应用，与开发工作流无缝集成。

### 2.3 最佳实践建议

#### 推荐采用

1. **代码语法高亮**（来自 Typora/Office Viewer）
   - 所有竞品都支持，属于基础功能
   - 使用 `@milkdown/plugin-prism` 或 `@milkdown/plugin-shiki`

2. **Mermaid 图表支持**（来自 Typora/Office Viewer）
   - 技术文档常用功能
   - 使用 `@milkdown/plugin-diagram`

3. **大纲视图**（来自 Typora/Office Viewer）
   - 长文档导航必备
   - 基于 ProseMirror 文档结构实现

4. **撤销/重做按钮**（来自 Office Viewer）
   - 非技术用户友好
   - history 插件已有，只需添加 UI

5. **字数统计**（来自 Typora/Office Viewer）
   - 写作者常用功能
   - 实现简单

#### 建议避免

1. **复杂的图片编辑功能**
   - Typora 有但实现复杂
   - 建议后期再做

2. **Word 导出**
   - 实现复杂，依赖 Pandoc
   - 建议后期再做

#### 创新机会

1. **VSCode 原生集成优势**
   - Git diff 集成
   - 多光标编辑
   - 命令面板集成
   - 工作区文件关联

2. **AI 辅助写作**
   - 集成 GitHub Copilot
   - 智能续写
   - 语法检查

### 3.2 用户故事

#### US-001: 代码语法高亮

**格式**: 作为技术写作者，我想在代码块中看到语法高亮，以便更清晰地展示代码示例。

**详细描述**:
- **角色**: 技术写作者
- **功能**: 代码块根据语言自动高亮显示
- **价值**: 提高代码可读性，提升文档质量
- **对标参考**: Typora 使用 CodeMirror 实现高亮

**验收标准**:
- AC-001-01: 支持 100+ 种编程语言的语法高亮
- AC-001-02: 支持 ```language 语法指定语言
- AC-001-03: 高亮样式与 VSCode 主题一致
- AC-001-04: 行号显示可选

**边界条件**:
- 输入: 代码块内容 + 语言标识
- 输出: 带语法高亮的渲染结果
- 异常: 未知语言使用纯文本显示
