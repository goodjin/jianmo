# M8 验收用文档场景（手工 / 半自动）

> 不强制把超大 `.md` 进仓库；本地可用脚本生成。下列阈值对应 `webview/src/utils/richPerfTier.ts` 的 **0 / 1 / 2** 档。

## 场景 S：小文档（档 0）

- 规模示例：&lt; 1 万字符，&lt; 500 行
- 期望：开启 Shiki（若配置为 true）与 Mermaid；无明显启动等待

## 场景 M：中等（档 1）

- 规模示例：约 100k 字符 或 2500 行（不跨档 2）
- 期望：首屏可尽快输入；Mermaid 仅滚动到视口时逐步渲染；Shiki 关闭或运行时关闭；顶部出现「中等规模」类提示

## 场景 L：大文档（档 2）

- 规模示例：&gt; 180k 字符 或 &gt; 4000 行
- 期望：不加载 Mermaid/不重跑 Shiki；有明确降级提示；仍可编辑与保存
- 可选：点击「仍要完整渲染」后，确认动画/输入可能变卡（主观）

## 基线命令

- `npm run build:webview && npm run m8:baseline`：输出 `dist/webview` 规模统计，发版前对比
