# 遥测：dashboard 规划（可选）（M338）

## 前提

- 默认 **不启用遥测**，且不采集用户内容（见 `docs/TELEMETRY_PRIVACY_REVIEW_M323.md`）
- 本规划仅定义“如果未来要做 dashboard，应如何做才不踩隐私红线”

## 目标（只看健康度）

- **稳定性**：启动失败率、保存失败率（`SAVE_FAILED`）、导出失败率（按错误类型聚合）
- **性能**：init 时间分布（P50/P95）、大文档操作耗时分布（聚合）
- **兼容性**：协议不兼容/降级次数（聚合）

## 数据最小化原则

- 仅上报 **计数/直方图**，不包含文件名、路径、正文、图片列表等敏感信息
- 用户可一键关闭；默认关闭

## Dashboard 形态（建议）

- 先本地/CI 产出：`gates:stable` 输出聚合摘要（不联网）
- 若未来接入远端：
  - 仅接入“聚合后指标”
  - 严格文档化字段、保留期、访问权限
