## M7：性能与包体瘦身（Performance + Bundle）

> 状态：已完成（2026-04-22）
> 目标版本：`1.4.4`（PATCH）

### 1) 背景与问题

当前 VSIX 包含的 webview 构建产物文件数与体积偏大（尤其是语法高亮/渲染相关资源），会带来：

- 安装/升级更慢
- 首次打开 webview 更慢
- 运行时内存与 CPU 压力增大

### 2) 目标（DoD）

- **包体更小**：VSIX 体积显著下降（目标：较当前基线下降 ≥ 30%）。
- **文件更少**：webview 产物文件数明显减少（目标：较当前基线下降 ≥ 50%）。
- **启动更快**：Rich 首次进入/切换更快（目标：主观体验明显改善；后续可补量化埋点）。
- **稳定性不退化**：重渲染能力（Mermaid/Shiki）仍可用，失败时可降级，不影响编辑。
- **门禁可守**：新增“产物大小/文件数”守护脚本或测试阈值。

### 3) 任务顺序（按依赖）

1. **M7-1 打包瘦身**（最高优先级）
2. **M7-2 缓存与惰性渲染**
3. **M7-3 大文档降级策略**
4. **M7-4 包体/产物守护门禁**
5. **M7-5 发版 1.4.4**

### 6) 执行记录

- [x] **M7-1**：合并 shiki/mermaid vendor chunk，显著降低 webview 产物文件数；Shiki 改为启用时动态加载
- [x] **M7-2**：Mermaid/Shiki 增加缓存，减少重复渲染/高亮开销
- [x] **M7-3**：大文档自动降级（关闭部分重渲染）并提示
- [x] **M7-4**：新增包体守护脚本 `npm run check:bundle`
- [x] **M7-5**：发版 `1.4.4`（门禁/构建/打包）

### 4) M7-1：打包瘦身（范围与验收）

#### 4.1 策略

- **Shiki**：
  - 运行时按需加载（语言/主题/wasm 不进主包或尽量拆成独立 chunk）
  - 默认不把全量语言包打进 `dist/webview`
- **Mermaid**：
  - 保持懒加载；必要时进一步拆图类型模块
- **产物排除**：
  - 检查 `dist/webview` 是否存在可排除的非必要文件（例如 debug 资源）
  - 若需要，通过 `.vscodeignore` 或构建配置排除不必要产物

#### 4.2 验收

- `npm test` 全绿
- `npm run build` 全绿
- `vsce package` 后：
  - VSIX 体积下降
  - VSIX 内 `dist/webview` 文件数明显减少

### 5) M7-4：守护门禁（建议）

- 新增脚本：统计 `dist/webview` 文件数/总大小/最大 chunk
- 阈值可先设置为“回归不变大”，后续逐步收紧
