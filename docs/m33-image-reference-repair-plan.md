# M33 图片引用修复与资产维护计划

目标：在 M32 已能检测缺失图片的基础上，让用户可以复制缺失清单、打开资源目录、选择新图片修复引用，并批量规范化本地图片路径。

## M33-0：计划与验收标准

- [x] 新建本计划文档
- [x] 明确首批范围：命令面板触发，不新增复杂 UI
- [x] 验收：缺失图片不只可发现，还能复制、定位和修复

## M33-1：复制缺失图片清单

- [x] 新增图片资产命令：复制缺失图片清单
- [x] 复制前复用 M32 的本地图片检测结果刷新
- [x] 输出包含原始引用、解析路径和错误信息

## M33-2：打开文档目录或 assets 目录

- [x] 新增图片资产命令：打开 assets 目录
- [x] Extension 侧按当前文档目录和 `image.saveDirectory` 定位目录
- [x] 验收：不依赖用户手动查找资源目录

## M33-3：选择新路径并替换图片引用

- [x] 新增图片资产命令：修复第一个缺失图片引用
- [x] Extension 侧弹出图片选择框，并返回相对当前文档的新路径
- [x] Webview 侧精确替换 Markdown 图片引用中的旧 `src`

## M33-4：批量规范化本地图片路径

- [x] 新增图片资产命令：规范化本地图片路径
- [x] 将本地图片路径统一为 `image.saveDirectory + basename`
- [x] 远程图、data URL、绝对路径不参与改写
