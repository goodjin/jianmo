# A11y：键盘可达与 aria 基线（M308）

## 目标

建立一个“不会退化”的无障碍基线，覆盖最常用的面板与关键操作：

- 工具栏（模式切换、格式化、导出）
- 查找替换面板
- 大纲面板
- 关键对话框（润色预览、表格转换预览、标题建议、资产面板等）

## 基线清单（当前仓库现状）

- **语义角色**：多个面板使用 `role="region"` / `role="search"`；弹层使用 `role="dialog"` + `aria-modal="true"`。
- **可感知状态**：toast / banner 使用 `role="status"` + `aria-live="polite"`。
- **键盘可达**：
  - 工具栏模式切换按钮使用 `tabindex` 与 `@keydown`，避免焦点丢失。
  - 大纲筛选输入支持 `Esc` 清空。

## 建议的“必须守住”的规则（后续实现/审查按此执行）

1. 任何可点击元素必须可用键盘触发（Enter/Space）。
2. 所有 dialog 必须具备可感知标题（`aria-label` 或 `aria-labelledby`）。
3. 动态提示（toast/status）必须有 `aria-live`（避免屏幕阅读器静默）。
4. 面板标题与按钮 `aria-label` 不得为空，且不要仅靠图标表达。

## 后续工作

本基线文档先行落地；下一步会在 M309–M311 中补齐：

- 对比度与主题变量一致性
- 更系统的屏幕阅读器标签与交互测试流程

# A11y：键盘可达与 aria 基线（M308）

## 目标

建立最小可用的无障碍基线：

- 重点面板/按钮/对话框 **可用键盘操作**
- 对话框打开后 **焦点可进入**，并且 **不会跑出弹窗**
- 常见 `aria-*` 缺失导致屏幕阅读器“读不出”的问题尽量补齐

## 本次落地（Webview）

### 1) Modal 对话框键盘支持

在 `webview/src/App.vue`：

- 当任意 modal 打开时：
  - **Esc** 关闭最上层 modal
  - **Tab / Shift+Tab** 在 modal 内形成 **焦点环**（focus trap）
  - 打开 modal 会把焦点移动到弹窗内的第一个可聚焦元素，并在关闭后恢复到打开前的元素（best-effort）

覆盖的弹窗：

- Rich 表格快捷键帮助
- AI 润色预览
- 选区转表预览
- AI 操作回看
- 标题建议
- 图片资产面板

### 2) aria 基线

现有结构已包含：

- 主要区域的 `role` / `aria-label`
- toast `aria-live="polite"`
- 多个弹窗 `role="dialog"` + `aria-modal="true"`

## 后续（M309+）

- 对各子组件补齐更细的 aria（按钮/列表/输入框）
- 为屏幕阅读器场景补充更明确的可读文本与状态变化播报
