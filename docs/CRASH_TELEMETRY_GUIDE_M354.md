# 可靠性：崩溃/异常捕获口径（可选）（M354）

## 结论（本阶段）

**不引入远端崩溃遥测**，只固化“本地诊断 + 复现信息”口径。原因：隐私与合规成本高；且当前自救中心已能承载大部分问题定位。

## 口径

- **崩溃定义**：
  - Webview 白屏/红屏无法恢复
  - 保存/导出链路异常导致功能不可用
- **收集方式（用户）**
  1. 重载窗口
  2. 打开自救中心复制诊断信息
  3. 提供复现步骤与环境（Remote/WSL、OS、VS Code 版本）

## 后续如果要做远端

- 只上报聚合计数/直方图，不包含内容
- 默认关闭，用户显式开启
- 严格字段白名单（见 `docs/TELEMETRY_PRIVACY_REVIEW_M323.md`）
