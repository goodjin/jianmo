# UX：快捷键冲突检测与 FAQ（M329）

## 目标

当用户反馈“快捷键不生效/被其它扩展抢走”时，有一份可直接转发的 FAQ 和自救步骤。

## 常见现象

- `Cmd+\` / `Ctrl+\` 无法切换模式
- `Ctrl+F` 打开的是 VS Code 内置查找，而不是 Markly 面板（或相反）

## 自救步骤（用户版）

1. 打开 VS Code **键盘快捷方式**（Keyboard Shortcuts）
2. 搜索 `markly.`，找到对应命令（例如 `markly.toggleMode`）
3. 观察是否有冲突提示，并改成你喜欢的按键

## 开发者注意

- 任何新增快捷键都应：
  - 尽量设置合理的 `when`（避免在非 markdown 场景抢键）
  - 在 README 或文档里补一条可发现的入口
