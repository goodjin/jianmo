# Image assets：大批量资源性能与 UX（M335）

## 场景

用户在 `./assets/` 或工作区里有大量图片（数千张），典型痛点：

- 扫描慢
- “未引用资源”清理耗时高
- 诊断信息太大（复制卡顿）

## 策略

- **扫描分页**：一次最多处理 N 条路径（如 200），分批汇总
- **结果截断**：未引用资源列表最多展示前 200 条；超出显示“已截断”
- **缓存**：
  - 基于文件 mtime 的轻缓存（避免重复全扫）
  - 切换文档只做增量更新
- **诊断瘦身**：
  - 诊断中只放统计与前 K 个样例路径
  - 完整列表通过“导出诊断文件”（可选未来增强）

## 用户提示（UX）

- 扫描时明确显示“正在扫描（x/y）”
- 超过预算时提示“已降级为部分结果”，并给出下一步（缩小范围/使用 VS Code 搜索）
