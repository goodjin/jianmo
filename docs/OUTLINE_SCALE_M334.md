# Outline：大文档性能与折叠策略（M334）

## 目标

在“超大 Markdown（上万行）”场景下：

- 大纲仍可用（哪怕是降级可用）
- 折叠/跳转不导致明显卡顿或崩溃

## 策略

- **解析策略**：
  - 优先做“轻量 heading 扫描”（只提取 `#` 层级、行号、标题文本）
  - 避免全量 AST（除非在后台线程/可缓存）
- **刷新策略**：
  - 输入时延迟更新（debounce）
  - 只在“停顿”或“保存/切换文档”时做一次完整刷新
- **UI 截断**：
  - 最多展示前 **500** 个节点
  - 超出时提示“已截断”，并提供搜索过滤
- **折叠策略**：
  - 默认仅折叠到 2–3 级，避免深层树导致渲染压力
  - 提供“一键全部展开/折叠”

## 失败/降级

- 解析耗时超过预算（如 200ms）：回落到“仅显示最高层级 heading”
- 将耗时与降级原因写入诊断（便于 issue 复现）
